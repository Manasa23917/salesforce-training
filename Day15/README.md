# Day 15 – Data Management & Data Quality

# Task 1 – Bad Data Scenarios

| Bad Data Problem | Business Problem |
|-----------------|----------------|
| Duplicate student records | Multiple records for same student |
| Missing email address | Notifications cannot be sent |
| Wrong department assigned | Incorrect reporting |
| Invalid attendance values | Wrong eligibility calculations |
| Duplicate course allocation | Scheduling conflicts |
| Incorrect phone numbers | Communication failures |
| Missing student ID | Tracking becomes difficult |
| Wrong fee information | Billing errors |
| Invalid dates | Incorrect records |
| Empty required fields | Incomplete information |

---

# Task 2 – Data Migration Thinking

Moving from Excel sheets to Salesforce may create challenges such as:

- Duplicate records
- Missing information
- Different date formats
- Incorrect field mapping
- Invalid records
- Inconsistent data formats

Migration requires cleaning and validating data before import.

---

# Task 3 – Data Governance Reflection

Clean and reliable data is important because:

- improves accuracy
- supports better decisions
- reduces errors
- improves reporting
- increases reliability

Bad data creates business problems.

---

# Task 4 – Enterprise Thinking

If 50,000 student records are imported incorrectly:

Problems may include:

- Wrong notifications sent
- Incorrect attendance records
- Fee calculation errors
- Reporting mistakes
- Duplicate student accounts
- Missing student information
- Poor decision making

Large data errors affect entire organizations.

---

# Duplicate Prevention Ideas

- Use validation rules
- Require unique IDs
- Check duplicates before import
- Validate CSV files
- Review data before migration

---

# Revision Questions

### 1. Why is clean data important?

Because accurate data improves reliability and decision making.

### 2. What problems happen because of duplicate records?

Duplicate records create confusion and incorrect reporting.

### 3. Why is data migration difficult?

Because data may contain duplicates, missing values, and inconsistent formats.

### 4. What is Data Loader used for?

Data Loader is used for importing, exporting, updating, and deleting data.

### 5. Why should enterprises validate imported data?

To prevent incorrect records from entering systems.

### 6. Why are CSV formats important?

Because they help organize data for import/export.

### 7. What risks happen during bulk import?

Duplicates, incorrect mapping, and data errors.

### 8. Why is governance important in data management?

Governance ensures data quality and reliability.
