# Day 14 – Flow Logic & Governance

## Task 1 – Approval Workflow Examples

### Course Creation
Faculty Request → Department Head → Academic Admin → Course Created

If rejected:
Request sent back.

### Faculty Leave Request
Faculty → Department Head → HR → Approved

If rejected:
Leave denied.

### Student Scholarship Request
Student → Department Verification → Scholarship Team → Finance Approval

If rejected:
Student notified.

### Budget Approval
Department → Manager → Finance → Admin Approval

If rejected:
Returned for review.

# Task 2 – Branching Flow Logic

Attendance Check:

If attendance <75%

→ Send warning email

If attendance <60%

→ Notify parents

If attendance <50%

→ Escalate to admin

Decision points:
- Attendance percentage checked
- Correct branch selected

Actions:
- Emails
- Notifications
- Escalation

# Task 3 – Governance Thinking

Enterprise systems cannot allow everyone to change records because:

- Security risks
- Wrong approvals
- Misuse of data
- Business risks

Controlled access protects systems.

# Task 4 – Reflection

Controlled workflows are needed because:

- Reduce mistakes

# Revision Questions

### 1. Why are approval workflows important?

They ensure proper authorization before actions happen.

### 2. Why do businesses require governance?

To maintain control, security, and accountability.

### 3. What are branching workflows?

Workflows that follow different paths based on conditions.

### 4. Why should automation follow business rules?

To ensure correct and safe operations.

### 5. Why are decision nodes important in flows?

They determine which path automation follows.

### 6. Why should enterprises restrict sensitive operations?

To avoid misuse and protect important data.

### 7. Why are approvals important in large organizations?

They ensure decisions are verified before execution.

### 8. Why should workflows be auditable?

To track actions and maintain transparency.
