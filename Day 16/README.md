# Day 16 – Debugging & Best Practices

# Task 1 – Bug Analysis

### Duplicate Notifications

Debugging approach:

- Check automation rules and flows
- Verify duplicate triggers
- Review debug logs
- Confirm notification logic

### Wrong Attendance Calculations

Debugging approach:

- Check calculation formulas
- Verify stored attendance values
- Test with sample data
- Review logs

### Flow Not Triggering

Debugging approach:

- Verify flow activation
- Check entry conditions
- Review debug logs
- Test flow manually

### Approval Process Stuck

Debugging approach:

- Check approval rules
- Verify permissions
- Review process steps
- Analyze logs

# Task 2 – Performance Thinking

If 50,000 users use the system simultaneously:

### UI Problems

- Slow loading pages
- Delayed responses
- Poor user experience

### Backend Problems

- High server load
- Slow processing
- System bottlenecks

### Database Problems

- Slow queries
- Data retrieval delays
- Increased load

### Notification Problems

- Delayed messages
- Failed notifications
- Queue buildup

### Automation Problems

- Slow workflows
- Delayed execution
- Processing failures

---

# Task 3 – Maintainability Thinking

Developers should write:

### Modular Code

Because smaller modules are easier to maintain.

### Reusable Components

Because code can be reused multiple times.

### Debuggable Systems

Because issues become easier to identify and fix.

Quick hacks create future problems.

---

# Task 4 – Reflection

Debugging is important because:

- software always has problems
- bugs affect users
- debugging improves reliability
- helps developers find root causes
- keeps systems stable

Debugging is essential for software engineering.

---

# LWC Best Practices

- Build reusable components
- Keep components small
- Avoid duplicate code
- Improve performance
- Write maintainable code

# Revision Questions

### 1. Why are debug logs important?

They help identify errors and understand system behavior.

### 2. Why is debugging difficult in enterprise systems?

Because many systems, users, and integrations interact together.

### 3. What problems happen when systems scale?

Performance issues, delays, and increased failures.

### 4. Why should components be reusable?

To reduce duplicate work and improve maintainability.

### 5. Why is maintainability important?

Because systems need updates and fixes over time.

### 6. Why should developers avoid tightly coupled code?

Because changes become difficult and risky.

### 7. Why do enterprise systems require monitoring?

To detect problems quickly and maintain reliability.

### 8. Why is troubleshooting an important engineering skill?

Because engineers must solve problems efficiently.
