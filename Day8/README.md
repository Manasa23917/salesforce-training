# Day 8 - LWC Basics

## What is LWC?

LWC stands for Lightning Web Components. It is a modern Salesforce UI framework used to build fast, reusable, and interactive components using HTML, JavaScript, and XML.

---

# Why Salesforce Uses LWC

Salesforce uses LWC because it is faster, reusable, secure, and follows modern web standards. It helps developers create better user interfaces for enterprise applications.

---

# Why Reusable Components Are Useful

Reusable components reduce duplicate work, improve maintenance, save development time, and create a consistent UI across the application.

---

# Frontend vs Backend Logic

## Frontend / UI Logic

- Button click actions
- Showing notifications
- Form display
- User interaction

---

## Backend / Apex Logic

- Fee calculation
- Database operations
- Complex validation
- External integrations

---

# Reflection

Modern enterprise systems use component-based UI architecture because it improves reusability, scalability, maintenance, and development speed.

---

# Revision Questions

## 1. What is a component?

A component is a reusable part of a user interface.

---

## 2. Why are reusable components useful?

They save time, reduce duplicate code, and improve maintenance.

---

## 3. Difference between frontend and backend?

Frontend handles UI and user interaction, while backend handles business logic and data processing.

---

## 4. Why did Salesforce move toward LWC?

Salesforce moved to LWC for better performance and modern web development.

---

## 5. Why is UI important in enterprise systems?

Good UI improves user experience and productivity.

---

## 6. Why should systems separate UI and business logic?

It improves security, maintenance, and scalability.

---

## 7. What security risks exist in enterprise applications?

Unauthorized access, data leaks, and insecure APIs.

---

## 8. Why should developers think modularly?

Modular design improves reuse, scalability, and easier maintenance.
